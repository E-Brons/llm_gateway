# Release Notes — v0.1.0

**Date:** 2026-04-02
**Status:** Initial release

---

## Overview

First public release of **llm_gateway** — a general-purpose Python library and local HTTP gateway that unifies access to multiple LLM backends under a single typed interface. Designed to decouple application code from provider specifics, routing configuration, and retry logic.

---

## What's Included

### Six Typed LLM Interfaces

Each interface defines a fixed input/output contract. Application code depends only on the abstract type — not on the backend.

| Interface | Method | Output |
|-----------|--------|--------|
| `GeneralLLM` | `complete(messages)` | `TextResponse` |
| `TextGenLLM` | `complete(messages, max_retries)` | `TextResponse` |
| `ReasoningLLM` | `complete(messages, thinking_budget?)` | `TextResponse` |
| `ImageGenLLM` | `generate(prompt, ...)` | `ImageResponse` |
| `ImageInspectorLLM` | `inspect(image, system, prompt)` | `TextResponse` |
| `ToolsLLM` | `complete(messages, tools)` | `ToolCallResponse` |

### Three Backends

| Backend | Key | Notes |
|---------|-----|-------|
| **Ollama** | `implementation: ollama` | Local REST API, no API key |
| **LiteLLM** | `implementation: litellm` | Cloud APIs (Anthropic, OpenAI, Gemini, …) via env var keys |
| **Claude CLI** | `implementation: cli` | `claude` subprocess; auth via `~/.claude/`. Supports General, TextGen, Reasoning, ImageInspector — not ImageGen or Tools |

### Local HTTP Gateway (FastAPI)

Run the gateway as a local server to expose all interfaces over HTTP:

```sh
./scripts/run.sh               # starts on port 4096
./scripts/run.sh --config port:9191
```

Endpoints: `GET /health`, `GET /models`, `POST /general`, `/text_gen`, `/reasoning`, `/image_gen`, `/image_inspector`, `/tools`

Interactive API docs auto-generated at `/docs` (Swagger) and `/redoc`.

### Configuration

Two-file configuration system with deep-merge local overrides:

| File | Purpose | Tracked |
|------|---------|---------|
| `llm_route.yml` | LLM routing — which model/backend handles each task | ✅ Yes |
| `settings.json` | Server behaviour — port, timeouts, concurrency, logging | ✅ Yes |
| `local/llm_route.yml` | User routing overrides | ❌ Gitignored |
| `local/settings.json` | User server overrides | ❌ Gitignored |

Local files are deep-merged at startup — only the keys you want to change need to be specified.

### Key Settings

- **Port**: `4096` (context window size — intentional)
- **Timeout**: `600 s` global ceiling
- **Concurrency**: Ollama `3`, Claude CLI `5`, LiteLLM `5`
- **Logging**: structured startup log showing resolved config + settings
- **Log path**: strftime format, e.g. `/tmp/%Y-%m-%d_%H-%M-server-log`

## Developer Experience

- **`pip install -e ".[dev]"`** — installs runtime + pytest + ruff
- **`./scripts/test.sh`** — runs lint (`ruff check` + `ruff format --check`) then all unit tests
- **CI** — GitHub Actions: lint → test (Python 3.14, coverage ≥ 85%)
- **Pre-push hook** — blocks push if ruff check or format fails (activated by `./scripts/install.sh`)
- **109 unit tests**, **95% coverage** — all mocked, no services required

## Known Limitations

- `ToolsLLM` is not implemented for the CLI backend (Claude CLI has no tool-call protocol)
- `ImageGenLLM` is not implemented for the CLI backend
- `settings.json` values (port, timeout, concurrency, defaults) are loaded and logged at startup but not yet applied at the call level — enforcement is planned for v0.2.0
- No streaming support — all calls are blocking (`stream: false`)
- No authentication on the HTTP gateway — designed for local use only (`allow_external_port: false` default)
