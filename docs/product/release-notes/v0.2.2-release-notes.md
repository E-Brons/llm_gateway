# Release Notes — v0.2.2

**Date:** 2026-04-11
**Status:** Release

---

## Overview

v0.2.2 extends the `ipadapter` and `ipadapter_faceid` endpoints with the full set of standard diffusion parameters: negative prompt, CFG scale, and LoRA support. The IP-Adapter conditioning strength parameter is also renamed from the generic `weight` to the domain-specific `ip_adapter_scale` for clarity.

---

## What's New

### New parameters: `negative_prompt`, `cfg_scale`, `lora`, `lora_weight`

All four parameters are available on both `/ipadapter` and `/ipadapter_faceid` endpoints, the gateway implementations, and the pipeline functions.

| Parameter | Type | Default | Description |
|---|---|---|---|
| `negative_prompt` | `str \| None` | `None` | Text describing what to avoid in the generated image |
| `cfg_scale` | `float` | `7.5` | Classifier-free guidance scale — higher values follow the prompt more strictly |
| `lora` | `str \| None` | `None` | HuggingFace repo ID or local path to LoRA weights |
| `lora_weight` | `float` | `1.0` | LoRA conditioning strength (0–1); only applied when `lora` is set |

**Usage via gateway:**

```python
llm = factory.ipadapter_faceid()
result = llm.generate(
    "a person in a space suit, cinematic lighting",
    reference_images=[face_bytes],
    ip_adapter_scale=0.75,
    negative_prompt="cartoon, anime, blurry",
    cfg_scale=9.0,
    lora="civitai/my-style-lora",
    lora_weight=0.6,
    seed=42,
)
```

**LoRA behaviour**: LoRA weights are loaded per-request inside the inference lock and unloaded immediately after — the pipeline remains stateless between calls. Both HuggingFace repo IDs and local file paths are accepted.

---

### Rename: `weight` → `ip_adapter_scale`

The `weight` parameter on `ImageGenLLM.generate()` and in the HTTP request payloads is renamed to `ip_adapter_scale` to match the IP-Adapter domain terminology and make its purpose unambiguous.

**Affected surfaces:**

- `ImageGenLLM.generate()` abstract base method
- `DiffusionServerIPAdapterLLM.generate()`
- `DiffusionServerIPAdapterFaceIDLLM.generate()`
- `OllamaImageGenLLM.generate()` (accepts and ignores it, as before)
- `POST /ipadapter` and `POST /ipadapter_faceid` request bodies
- `generate_ipadapter()` and `generate_ipadapter_faceid()` pipeline functions

---

## Breaking Changes

**`weight` → `ip_adapter_scale`** on `ImageGenLLM.generate()` and the HTTP API.

Callers passing `weight=...` must update to `ip_adapter_scale=...`. The default value (0.5) is unchanged.

---

## Internal Improvements

- `negative_prompt` and `cfg_scale` are forwarded to the diffusers pipeline `guidance_scale` and `negative_prompt` arguments respectively, enabling proper negative conditioning in all generation calls.
- Optional params (`negative_prompt`, `cfg_scale`, `lora`, `lora_weight`) are omitted from the HTTP payload when not set, keeping requests minimal.

---

## Test Coverage

- Updated existing param-forwarding tests for the `weight` → `ip_adapter_scale` rename.
- New unit tests: optional params omitted when `None`, `cfg_scale` forwarded when set, `lora` + `lora_weight` forwarded together, default `ip_adapter_scale` is 0.5.
- Total unit test count: **~168 tests**, all passing.
