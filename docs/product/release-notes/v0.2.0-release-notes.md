# Release Notes — v0.2.0

**Date:** 2026-04-08
**Status:** Release

---

## Overview

v0.2.0 adds two new image generation routes powered by IP-Adapter, introduces per-request overrides for temperature and structured output, and cleans up the gateway type model so every route uses a standard abstract type.

---

## What's New

### IP-Adapter Image Generation (ipadapter, ipadapter_faceid)

Two new optional routes for conditioned image generation via a local diffusion pipeline server:

| Route | Purpose | Factory method |
|-------|---------|---------------|
| `ipadapter` | Style-transfer: generate an image guided by a reference style image | `factory.ipadapter()` |
| `ipadapter_faceid` | Portrait synthesis: generate an image that retains the identity of a face image | `factory.ipadapter_faceid()` |

Both routes return a standard `ImageGenLLM` (the same abstract type as `image_gen`) and an `ImageResponse`.  No new types are required in application code.

**Usage**:

```python
factory = create_factory("llm_route.yml")

# Style-conditioned generation
llm = factory.ipadapter()
result = llm.generate(
    "A cat in the style of the reference image",
    reference_images=[open("style_ref.png", "rb").read()],
    weight=0.6,           # adapter conditioning strength (default 0.5)
    width=512,
    height=512,
    seed=42,
)

# Face-conditioned generation
llm = factory.ipadapter_faceid()
result = llm.generate(
    "A person in a space suit, cinematic lighting",
    reference_images=[open("face.png", "rb").read()],
    weight=0.75,
    width=512,
    height=512,
)
```

**Configuration** (`llm_route.yml`):

```yaml
ipadapter:
  implementation: diffusion_server
  model: ip-adapter_sd15_light_v11
  api_base: http://localhost:7860

ipadapter_faceid:
  implementation: diffusion_server
  model: ip-adapter-faceid-plus_sd15
  api_base: http://localhost:7860
```

Both routes are **optional** — omitting them from `llm_route.yml` is valid.  Calling `factory.ipadapter()` without configuration raises `ValueError` with a clear message.

**Prerequisites**: A diffusion pipeline server must be running at `api_base`.  The server exposes `POST /ipadapter` and `POST /ipadapter_faceid` endpoints (see software design doc §7.4 for payload specification).

### weight Parameter on ImageGenLLM

The `weight` parameter is now a first-class argument on `ImageGenLLM.generate()`:

```python
llm.generate(prompt, reference_images=[ref], weight=0.8)
```

`weight` controls adapter conditioning strength.  It defaults to `None` (no effect on backends that ignore it) and is used only by the diffusion server implementations where it defaults to `0.5`.  Ollama and LiteLLM image-gen backends accept and silently ignore it.

---

### Per-Request Temperature Override

All text completion types (`GeneralLLM`, `TextGenLLM`, `ReasoningLLM`, `ImageInspectorLLM`) now accept a `temperature` override directly on the `complete()` / `inspect()` call:

```python
result = llm.complete(messages, temperature=0.1)   # overrides constructor temperature
```

The per-call value takes precedence over the constructor-level `temperature`.  Passing `None` falls back to the constructor value.

---

### Per-Request Structured Output Override

All text-returning types now accept a `response_schema` override at call time:

```python
result = llm.complete(messages, response_schema={"type": "object", ...})
```

This supersedes the `response_schema` passed at construction time, enabling a single `LLM` instance to serve different output shapes without re-instantiation.

---

## Breaking Changes

None. The `ipadapter` and `ipadapter_faceid` config sections are optional; existing configs without them continue to work unchanged.

---

## Internal Improvements

- **Unified type model**: `ipadapter` and `ipadapter_faceid` routes now use `ImageGenLLM` as their abstract type (previously `IPAdapterLLM` / `IPAdapterFaceIDLLM` — separate non-standard types). Application code that typed against `ImageGenLLM` works with all three image-generation routes.
- `conftest.py`: fixed Python 2 `except A, B:` syntax to `except (A, B):`.

---

## Test Coverage

- Unit tests covering both diffusion server implementations: happy path, endpoint routing, payload structure, weight forwarding, model prefix stripping, retry on empty response, validator callback.
- Integration (sanity) tests for live diffusion server: `tests/test_ipadapter_sanity.py` — skipped automatically if the server is unreachable.
- Total unit test count: **~120 tests**, all passing.

---

## Compatibility

- **Python** ≥ 3.14
- **Diffusion server**: custom FastAPI server with IP-Adapter pipeline (not included — bring your own)
